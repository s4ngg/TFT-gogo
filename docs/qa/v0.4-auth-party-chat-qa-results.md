## Swagger/API 확인

`/v3/api-docs`에서 아래 path/tag 노출을 확인했다.

- Auth: `/api/v1/auth/signup`, `/api/v1/auth/login`, `/api/v1/auth/social/{provider}`
- Member: `/api/v1/members/me`
- Community Party: `/api/community/parties`, `/api/community/parties/{partyPostId}/join`
- Chat: `/api/community/chat/rooms/{roomId}/stream`
- Tags: `Auth`, `Member`, `Community Party`, `Chat`, `Riot Status` 등

## 확인된 수정 사항

- OAuth2 설정 없는 Docker/QA 환경에서 backend가 순환 참조로 죽던 문제는 #389에서 수정했다.
- OAuth client registration이 없을 때도 backend 기동과 Swagger 접근은 유지되고, OAuth browser filter chain만 조건부로 비활성화된다.
- 보호 API 인증 실패가 Spring Security 기본 403으로 떨어지던 문제는 #390에서 401 `ApiResponse`로 통일했다.
- 프론트 auth restore 흐름은 401을 인증 상태 정리 신호로 사용하므로 #390 방향과 일치한다.

## Known Risk / 후속 범위

- 전체 QA DB bootstrap은 #428에서 보강 중이다. #428 머지 전 `origin/develop` 기준 Docker local smoke DB에는 Auth/Party/Chat 전체 기본 schema가 자동 준비되지 않는다.
- 실제 OAuth provider flow는 client id/secret이 있는 QA 환경에서만 검증 가능하다.
- MVP 채팅은 메모리 저장이므로 서버 재시작 또는 멀티 인스턴스에서 메시지가 유지/공유되지 않는다.
- 파티 참여자만 채팅 가능하도록 제한하는 멤버십 검증과 `chat_rooms` row 영속화는 후속 범위다.
- owner close/delete 정책은 아직 별도 정의가 필요하다.
