# QA 진행 & 이슈/PR 워크플로우

> 작성일: 2026-06-24
> 대상: TFT-gogo 전체 페이지

---

## 1. QA 진행 방식

### 1-1. 준비

1. Claude에게 QA 대상 페이지 전달 (예: "덱모음에서 qa해볼만한거 있을까?")
2. Claude가 코드 기반으로 **행동 중심 QA 체크리스트** 제안
3. 체크리스트 보고 직접 페이지에서 테스트 진행

### 1-2. 발견 사항 전달

페이지에서 문제를 발견하면 아래 형식으로 전달:

```
패치노트 :

- 제외와 제거가 버튼처럼 보이고, 구분할 수 없음
- 이미지가 같이 제공되면 좋을듯함
- 주요변경에서 콜론 제거 필요 / 변경점 총 갯수 비표시 / 클릭 시 동작 없음
- 카테고리 버튼이 동작하지 않음
- 버튼관련한건 하나의 이슈로 묶기
```

포인트:
- 한 줄에 한 가지 문제/제안
- 관련 항목 묶을 때 마지막에 명시 (예: "버튼관련한건 하나의 이슈로 묶기")
- 스크린샷 필요 시 `docs/screenshots/`에 저장 후 파일명 전달

### 1-3. 스크린샷 활용

| 방법 | 설명 |
|------|------|
| `docs/screenshots/`에 저장 | Claude가 이미지 확인 → 이슈별 매칭 → 커밋/푸시 후 이슈에 이미지 태그 포함 |
| GitHub 드래그앤드롭 | `gh issue create --web`으로 브라우저에서 직접 첨부 (가장 간편) |

## 2. 이슈 생성

### 2-1. 흐름

```
QA 피드백 전달
  → Claude가 이슈 단위로 분류 & 목록 제시
  → 사용자 확인 (assignee, label 지정)
  → gh issue create 일괄 생성
```

### 2-2. 이슈 분류 기준

| 라벨 | 기준 | 예시 |
|------|------|------|
| `bug` | 동작하지 않거나 의도와 다르게 동작 | 카테고리 버튼 클릭 시 무반응 |
| `feature` | 없는 기능 추가 요청 | 변경사항에 이미지 제공 |
| `frontend` | 프론트엔드 관련 | UI/UX 개선 |
| `backend` | 백엔드 관련 | API 오류 처리 |

### 2-3. 이슈 본문 형식

```markdown
## 설명
(문제 또는 개선 요청 요약)

## 문제 목록        ← 여러 항목 묶을 때
### 1. 항목 제목
- 상세 설명

## 현재 동작
(지금 어떻게 되는지)

## 기대 동작
(어떻게 바뀌어야 하는지)
```

## 3. QA 발견 → 이슈 정리 문서

페이지별 QA 결과는 별도 문서로 기록:

| 파일 | 대상 |
|------|------|
| `qa-community.md` | 커뮤니티(파티/채팅) QA 발견 이슈 |
| (추가 시) `qa-{페이지명}.md` | 페이지별 QA 발견 이슈 |

문서 형식 (`qa-community.md` 참고):

```markdown
# {페이지명} QA 발견 이슈

> 확인일: YYYY-MM-DD
> 대상: `/{페이지경로}`

---

## 이슈 1. 제목

- 설명
- 제안: (있으면)
- ![스크린샷](screenshots/파일명.png)

---

## 이슈 2. 제목
...
```

## 4. 이슈 → PR 워크플로우

### 4-1. 브랜치 네이밍

```
fix/issue-{번호}-{간단설명}     ← bug
feat/issue-{번호}-{간단설명}    ← feature
```

### 4-2. PR 규칙

| 항목 | 규칙 |
|------|------|
| 제목 | `fix: 간단 설명 (#이슈번호)` 또는 `feat: 간단 설명 (#이슈번호)` |
| 본문 | Summary + Related Issue + Test plan |
| linked issue | `Closes #이슈번호` |
| assignee/label | 이슈와 동일 |

### 4-3. PR 본문 템플릿

```markdown
## Summary
- 변경 내용 요약

## Related Issue
Closes #이슈번호

## Test plan
- [ ] 테스트 항목 1
- [ ] 테스트 항목 2
```
