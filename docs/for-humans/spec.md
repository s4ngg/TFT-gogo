## 서비스 개요

TFT(전략적 팀 전투) 전적 검색 서비스.
소환사 전적 조회, 현재 메타 스냅샷, 덱 추천, 게임 가이드, 패치 노트를 제공한다.

---

## 페이지별 기능 정의

### 1. Dashboard — 메인 (소환사 전적 검색)

- 소환사명#태그 형식으로 검색할 수 있다
- 최근 검색어 태그를 클릭해 빠르게 재검색할 수 있다
- 검색 결과로 소환사 카드가 표시된다
  - 소환사명, 태그, 티어, LP, 승수, 패수, 승률
- 현재 패치 메타 스냅샷을 종합 / 상위권 / 마스터+ 탭으로 확인할 수 있다
  - 순위, 티어 등급(S/A+/A/B/C/D), 덱 이름, 주요 유닛, 승률, 평균 등수
- 소환사를 검색하지 않아도 메타 스냅샷은 기본 노출된다

### 2. Decks — 덱 목록 & 상세

#### 2-1. 덱 목록 (`/decks`)

- 현재 패치 추천 덱 목록을 확인할 수 있다
- 티어별 그룹핑으로 표시된다 (S / A+ / A / B / C / D)
- 랭크 구간 필터를 선택할 수 있다 (마스터+ / 다이아+ / 에메랄드+)
- 덱 이름: `[주요 시너지] [캐리 챔피언명]` 형식으로 자동 생성되며, 관리자가 직접 수정할 수 있다
- 각 덱에 챔피언 아이콘, 시너지 뱃지, 승률/TOP4/평균 등수/픽률을 표시한다
- 챔피언 목록은 해당 덱의 최종 운영 레벨 기준으로만 표시한다
  - 5코스트 캐리 덱 → Lv.9 기준 (5코스트 포함)
  - 4코스트 캐리 덱 → Lv.8 기준 (1~4코스트)
  - 3코스트 리롤 덱 → Lv.7 기준 (1~4코스트)
  - 1~2코스트 리롤 덱 → Lv.8 기준 (1~4코스트)

#### 2-2. 덱 상세 (`/decks/:deckId`)

- 추천 배치 (헥스 보드)
  - Lv.5~9 탭으로 레벨별 유닛 구성 확인
  - 레벨 탭 변경 시 배치판과 시너지 구성이 동시에 업데이트된다
  - 유닛은 코스트 오름차순으로 정렬 (빌드업 순서 반영)
  - 관리자가 배치판 포지션을 직접 지정한 경우, 자동 배치 대신 관리자 배치를 우선 표시한다
- 시너지 구성
  - 현재 선택 레벨의 visibleUnits 기반으로 계산 (CDragon 트레이트 데이터 활용)
  - Lv.6 기준, Lv.7 기준 등 레벨에 따라 동적으로 변경
- 추천 아이템
  - 캐리 유닛별 핵심 아이템 3개 표시
- 증강 추천
  - Riot API에서 증강 데이터를 제공하지 않음 (TFT-MATCH-V1 미지원 확인)
  - **향후 관리자 수동 큐레이션으로 제공 예정**

### 3. AiRecommend — AI 덱 추천

- AI가 현재 보유 유닛/아이템을 분석해 최적 덱을 추천한다
- 추천 덱은 유닛 구성, 아이템 배치, 포지셔닝을 포함한다

### 4. Guide — 게임 가이드

- 패치별 주요 변경사항을 확인할 수 있다
- 유닛/시너지/아이템 기본 정보를 조회할 수 있다

### 5. Party — 파티원 찾기

- 파티원 모집 글을 올리고 확인할 수 있다
  - 글 작성 시: 제목, 모집 인원, 티어 조건, 메모 입력
- 실시간 채팅 채널 목록을 확인하고 참여할 수 있다

### 6. PatchNotes — 패치 노트

- 최신 패치 노트를 확인할 수 있다
- 챔피언/아이템/시너지별 변경사항을 확인할 수 있다
- 패치 히스토리(이전 패치) 목록을 조회할 수 있다

### 7. Admin — 관리자 덱 큐레이션 (`/admin`)

> 자동 집계 데이터의 품질 보완을 위한 관리자 전용 페이지

- X-Admin-Token 헤더 기반 인증 (추후 ROLE_ADMIN으로 전환 예정)
- 덱 이름을 직접 수정할 수 있다 (자동 생성 이름 오버라이드)
- 중복/품질 낮은 덱을 숨길 수 있다 (데이터 삭제 아님)
- 덱 노출 순서를 수동으로 지정할 수 있다
- 추천 배치판 포지션을 헥스 그리드에서 챔피언별로 직접 지정할 수 있다
  - 미지정 시 CDragon range/role 데이터 기반 자동 배치 적용
- 큐레이션 데이터는 `signature` 기반으로 저장되어 재수집 후에도 유지된다

## 메타 덱 집계 정책

> Riot API에서 가져온 실매치 데이터를 집계해 메타 덱을 도출한다.

### 데이터 수집 한계

| 항목 | 내용 |
|------|------|
| Dev API 키 | 100 req / 2분 제한 |
| 집계 표본 | 수백~수천 게임 (Production 키 확보 전 제한적) |
| 증강 데이터 | **Riot API 미지원 — 자동 집계 불가** |
| 아이템 데이터 | 참여자별 최종 장착 아이템 기반 집계 |

### 아키타입 분류 기준

- **시그니처**: 상위 2개 트레이트 이름 조합
- **유사 덱 병합**: Jaccard 유사도 (코어 유닛 0.55 / 보드 유닛 0.65)
- **주요 캐리 기반 병합**: 동일 primaryCarryId + 보드 40% 이상 공유 시 병합
- **최소 표본**: 10게임 이상 등장한 덱만 노출
- **최소 선택률**: 0.5% 이상

### 큐레이션 레이어

자동 집계 한계를 보완하기 위해 관리자가 수동으로 다음을 설정할 수 있다:
- 덱 표시 이름
- 덱 숨김 여부 (중복 제거)
- 덱 노출 순서
- 추천 배치판 포지션 (헥스 그리드 기반, 미지정 시 자동 배치)

## 공통 기능

- 헤더: 로고, 주요 메뉴 네비게이션
- 사이드바: 메뉴 접기/펼치기, 현재 페이지 하이라이트
- Riot API 연동 상태: 현재 접속자 수, 대기열 표시
