# Layout Components

여러 페이지가 공유하는 앱 프레임 컴포넌트를 두는 위치입니다.

## 배치 기준

- `Sidebar`: 로고, 메인 메뉴, 랭크 카드, 의견 보내기처럼 전체 앱에서 유지되는 좌측 영역
- `TopBar`: 알림, 메일, 도움말, 프로필처럼 전체 앱 상단에서 유지되는 영역
- `AppLayout`: 페이지 본문을 감싸는 공통 shell이 필요할 때 사용

## 작업 규칙

- 페이지 전용 데이터나 화면 본문은 이 폴더로 올리지 않습니다.
- 메뉴 라벨, 경로, 프로필 표시처럼 여러 페이지가 공유하는 값은 layout 전용 데이터 파일로 분리합니다.
- 스타일은 `*.module.css`로 작성합니다.
- 팀원이 각자 페이지를 만들 때는 이 layout을 재사용하고, 본문만 `pages/<PageName>` 아래에 구현합니다.
