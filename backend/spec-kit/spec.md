# 메타 덱 집계 및 추천 스펙

이 문서는 덱모음, 메타통계, 덱 상세 추천 배치, 추천 아이템, 영웅 증강, 유물 아이템 추천의 기준을 정의한다.

## 집계 기준

- 메타 덱은 현재 패치 버전을 기준으로 집계한다.
- 패치 버전은 Riot 매치 상세 응답의 `game_version`에서 추출한다.
- 동일 패치와 동일 랭크 필터의 메타 덱을 다시 집계할 때는 기존 결과를 새 결과로 교체한다.
- 데이터 수집 기간은 집계 실행일 기준 최근 1일 단위 데이터를 기본으로 한다.
- 운영 환경에서는 메인 API 서버와 별도 수집 서버 또는 스케줄러 작업으로 집계를 수행할 수 있어야 한다.
- Riot API rate limit을 고려해 요청 간격과 재시도 정책을 적용해야 한다.
  - 현재 구현: 429 응답 시 고정 간격(`Thread.sleep`)으로 대기 후 재시도
  - 권장 개선안: 429 응답 헤더의 `Retry-After` 값을 파싱해 정확한 대기 시간 적용 (미구현, 로드맵 우선순위: 중간)

## 덱 중복 판단 기준

- 덱 이름 또는 대표 시너지명만으로 덱을 분리하지 않는다.
- 같은 아키타입의 덱은 핵심 유닛 유사도와 전체 보드 유닛 유사도를 기준으로 병합한다.
- 핵심 유닛은 고코스트, 2성 이상, 아이템 보유 유닛을 우선 고려한다.
- 진/블리츠크랭크처럼 일부 보조 유닛만 다른 경우에는 별도 덱이 아니라 같은 메타 덱 후보로 병합한다.
- 유사도 병합 후에도 표본 수가 부족한 덱은 메타 덱 노출 대상에서 제외할 수 있다.

> **구현 상태**: 기존 구현은 활성 특성 시그니처(상위 2개 트레잇 조합) 기준으로 덱을 분류한다. 현재 핵심 유닛/보드 유사도 기반 병합으로 전환 작업 중이며, 전환 전까지는 특성 시그니처가 같은 덱이 하나의 메타 덱으로 집계된다.

## 덱 노출 및 티어 기준

- 덱모음은 패치 기준 메타 덱을 선택률 중심으로 노출한다.
- 메타통계 화면은 순위 번호 대신 `S`, `A`, `B`, `C`, `D` 티어 그룹으로 구분한다.
- 티어는 전체 노출 덱의 상대 비율을 기준으로 나눈다.
  - `S`: 상위 10%
  - `A`: 다음 20%
  - `B`: 다음 30%
  - `C`: 다음 25%
  - `D`: 나머지
- 같은 선택률이면 평균 등수가 좋은 덱을 우선한다.

### 티어 분류 계산 규칙 (결정성 보장)

1. **정렬 기준**: 픽률(`pick_rate`) 내림차순 → 동률 시 평균 등수(`avg_rank`) 오름차순
2. **커트라인 계산**: 각 티어의 상한 인덱스 = `ceil(total_exposed_decks × tier_ratio)`
   - 예: 노출 덱 20개 기준 → S 2개, A 4개, B 6개, C 5개, D 나머지
3. **경계 동순위 처리**: 커트라인 직전·직후에 같은 (픽률, 평균 등수)를 가진 덱이 있으면 모두 높은 티어에 포함 (낮은 티어로 임의 절단 금지)
4. **최소 노출 수**: 패치당 집계된 메타 덱이 `MIN_EXPOSURE_COUNT`(기본값 5) 미만이면 티어 분류 없이 전체를 단일 목록으로 노출한다.

## 상점 구매 가능 유닛 기준

- 덱모음과 덱 상세에는 상점에서 구매 가능한 챔피언만 노출한다.
- 시너지, 스킬, 소환, 임시 효과로 생성되는 유닛은 노출하지 않는다.
- 상점 구매 가능 여부는 세트별 allowlist 또는 Community Dragon 정적 데이터를 기준으로 판정한다.
- 예외 유닛이 발견되면 집계 저장 단계와 API 응답 단계 모두에서 방어해야 한다.

## 추천 배치 기준

- 추천 배치는 최종 매치 결과를 그대로 복사하지 않고, 레벨별 추천 후보로 재구성한다.
- 레벨별 허용 코스트는 다음 기준을 따른다.
  - Lv.5~6: 3코스트 이하
  - Lv.7~8: 4코스트 이하
  - Lv.9: 5코스트 허용
- 4코스트와 5코스트 유닛은 추천 화면에서 3성 표기를 하지 않는다. 표시 별 등급은 최대 2성으로 제한한다.
- 챔피언의 사거리와 역할 정보는 Community Dragon 정적 데이터를 기준으로 사용한다.
- 사거리가 짧거나 탱커/브루저 역할인 유닛은 앞줄에 우선 배치한다.
- 사거리가 길거나 캐스터/마크스맨 역할인 유닛은 뒷줄에 우선 배치한다.
- 데이터가 부족해 특정 레벨의 권장 유닛 수를 채우지 못하면, 없는 유닛을 임의로 생성하지 않는다.

## 챔피언 추천 아이템 기준

- 한 덱에서 추천 아이템을 보유한 캐리 유닛은 최대 3명만 노출한다.
- 캐리 유닛별 추천 아이템은 일반 완성 아이템 최대 3개만 노출한다.
- 추천 아이템은 `마스터 이 > 밤의 끝자락, 거인 학살자, 수은`처럼 유닛 단위로 보여준다.
- 다음 아이템은 챔피언 추천 아이템에서 제외한다.
  - 조합 재료 아이템
  - `EmptyBag`
  - Radiant 아이템
  - Artifact/Ornn 아이템
  - Support 아이템
  - Emblem/Trait 아이템
  - Consumable/Temporary 아이템
- 유물/오른/특수 아이템은 챔피언 일반 추천 아이템이 아니라 별도 유물 추천 데이터로 집계한다.

## 영웅 증강 추천 기준

- 영웅 증강은 챔피언의 구조나 역할을 바꾸는 증강을 의미한다.
- 예시는 다음과 같다.
  - `침략자 제드`: 제드가 별도 핵심 유닛으로 등장하는 증강
  - `방패 여전사`: 레오나의 역할이 탱커에서 전사형으로 바뀌는 증강
- 영웅 증강은 일반 증강 전체 목록이 아니라, 메타 덱과 직접 연결되는 영웅형 증강만 추천 대상으로 삼는다.
- 표본이 적더라도 집계 결과가 있으면 추천 후보에 포함할 수 있으나, 화면에서는 추천/참고 상태를 구분한다.

## 유물 아이템 추천 기준

- 유물 아이템 추천은 유저가 유물 또는 오른 아이템을 획득했을 때 어떤 덱이나 유닛으로 이어갈지 안내하는 기능이다.
- 유물 아이템은 일반 챔피언 추천 아이템과 분리해 집계한다.
- 유물 아이템별 추천은 해당 아이템을 사용했을 때 성과가 좋은 덱과 장착 유닛을 함께 보여준다.
- 예시는 다음과 같다.
  - `에코의 인내` 획득 시 아우렐리온 솔, 조이 등 해당 아이템 효율이 높은 유닛과 덱 후보 추천

## 정적 데이터 기준

- Riot Match API는 매치 결과, 유닛, 아이템, 증강, 시너지 결과를 제공한다.
- 챔피언 이름, 코스트, 사거리, 역할, 스킬 설명 같은 정적 정보는 Community Dragon 데이터를 기준으로 보강한다.
- 프론트엔드는 Community Dragon CDN URL을 직접 하드코딩하지 않고 기존 asset helper를 통해 사용한다.
